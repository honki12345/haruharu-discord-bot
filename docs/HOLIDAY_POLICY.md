# 공휴일 정책

## 개요

기상 챌린지는 주말(토/일) 및 공휴일에도 오전 04:00 daily message와 출석 thread를 생성합니다.

- 주말/공휴일 13:00 집계는 결과 메시지를 공지하지 않습니다.
- 주말/공휴일도 출석 판정 창은 동일하게 적용되어, 등록 시간 기준 `+11분~12:59` 댓글은 `late`로 인정되고 `13:00` 집계 시점까지 무댓글이면 미참여로 남습니다.
- 주말/공휴일 `AttendanceLog.status='attended'` 또는 `AttendanceLog.status='late'` 는 `absencecount`를 우선 1회 차감하고, 차감할 결석이 없으면 `latecount`를 1회 차감합니다.
- 주말/공휴일 `attended`, `late` 댓글에는 기존 판정 이모지와 함께 `🎁` 반응을 추가하고, 둘 다 streak 증가 대상으로 유지합니다.
- 주말/공휴일 무댓글, `absent` 는 새 지각/결석 패널티를 추가하지 않습니다.
- 휴가 등록 사용자는 기존과 동일하게 휴가 우선 규칙을 유지합니다.

## 공휴일 정의 위치

`src/utils.ts` 파일의 `PUBLIC_HOLIDAYS_YYYY` 배열에 정의합니다.

```typescript
const PUBLIC_HOLIDAYS_2026 = [
  '0101', // 신정
  '0216', // 설날
  // ...
];
```

## 형식

- **MMDD**: 월(2자리) + 일(2자리)
- 예: 1월 1일 → `'0101'`, 12월 25일 → `'1225'`

## 포함해야 하는 날짜

| 구분        | 설명                                                                                         |
| ----------- | -------------------------------------------------------------------------------------------- |
| 법정 공휴일 | 신정, 설날, 삼일절, 어린이날, 부처님오신날, 현충일, 광복절, 추석, 개천절, 한글날, 크리스마스 |
| 대체공휴일  | 공휴일이 주말과 겹칠 경우 지정되는 대체휴일                                                  |
| 선거일      | 대통령선거, 국회의원선거, 지방선거 등                                                        |

## 연간 업데이트

매년 말(12월)에 다음 해 공휴일을 업데이트해야 합니다.

### 업데이트 절차

1. 다음 해 공휴일 목록 확인 (정부 발표)
2. `src/utils.ts`에서 기존 연도 배열을 주석 처리
3. 새 연도 배열 추가 (`PUBLIC_HOLIDAYS_YYYY`)
4. export 변경
5. `src/events/ready.ts`에서 import 및 사용 부분 변경
6. `src/test/utils.test.ts`에서 테스트 코드 변경
7. 이 문서의 예시 연도 업데이트

### 공휴일 확인 사이트

- [공공데이터포털 - 한국천문연구원 특일 정보](https://www.data.go.kr/data/15012690/openapi.do)
- [정부24 공휴일 안내](https://www.gov.kr)
