# ISSUE #106 구현 계획

## 제목

- [feat] 일일 출석표 문구 포맷 정리

## 배경

- 현재 일일 출석표 헤더가 `20260330 출석표`처럼 내부 키(`YYYYMMDD`) 형태로 노출돼 가독성이 떨어진다.
- 사용자별 상태 문구도 `월 누적`, `잔여휴가`처럼 길고 딱딱한 표현이 포함돼 한 줄 스캔이 어렵다.
- 현재 예시 출력은 아래와 같다.

```text
### 20260330 출석표
- 개나리: 출석 (월 누적 지각 0회, 결석 12회, 잔여휴가 5일)
- 김훈기: 출석 (월 누적 지각 0회, 결석 0회, 잔여휴가 5일)
- Akkim: 출석 (월 누적 지각 0회, 결석 0회, 잔여휴가 5일)
- Jue: 결석 (월 누적 지각 7회, 결석 9회, 잔여휴가 6일)
```

- 운영 관점에서는 정책이나 집계 로직을 바꾸는 작업이 아니라, 같은 정보를 더 읽기 쉬운 형태로 정리하는 출력 포맷 개선 작업이다.

## 변경안

- 헤더 날짜를 raw key 대신 사람이 읽기 쉬운 포맷으로 바꾼다.
  - 권장안: `### 2026-03-30 출석표`
- 괄호 안 문구에서 `월 누적`을 제거한다.
- `잔여휴가` 표기는 유지한다.
- 최종 행 포맷을 아래처럼 통일한다.

```text
- 개나리: 출석 (지각 0회, 결석 12회, 잔여휴가 5일)
```

## 구현 계획

상위 계층:

- 일일 출석표의 표시 전용 문자열 생성 규칙을 `src/services/reporting.ts`에서 정리한다.
- 내부 집계 키(`yearmonthday`)와 사용자 노출용 날짜 문자열을 분리한다.
- 상태 행은 동일한 수치 데이터를 유지하면서 라벨만 간결하게 재구성한다.
- 예시 기반 문서와 테스트를 함께 수정해 출력 포맷 회귀를 관리한다.

하위 계층:

- `src/services/reporting.ts`
  - 헤더 날짜 포맷터를 추가하거나 기존 날짜 문자열 조합을 사용자 표시용 형태로 분리한다.
  - `buildChallengeReportRow` 문자열에서 `월 누적`만 제거하고 `잔여휴가` 표기는 유지한다.
- `src/test/US-05-06-daily-attendance.test.ts`
  - `expectMonthlyStatus` 기대 문자열과 헤더 검증을 새 포맷으로 갱신한다.
- `docs/PROJECT.md`
  - 결과표 설명과 예시 문자열을 새 포맷으로 수정한다.
- `docs/USER_STORIES.md`
  - 일일 출석 리포트 시나리오의 결과표 예시를 새 포맷으로 수정한다.
